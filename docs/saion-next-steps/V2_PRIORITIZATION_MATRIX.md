# SAION V2 Prioritization Matrix

This matrix helps choose which V2 option to activate first when implementation is approved.

Scoring model (1 to 5):

1. User impact: value to user experience.
2. Build effort: implementation complexity and time (higher means more effort).
3. Delivery risk: uncertainty and dependency risk (higher means more risk).
4. Strategic fit: alignment with SAION's user-driven product direction.

Priority score:

Priority Score = User Impact + Strategic Fit - Build Effort - Delivery Risk

Higher score indicates a better near-term candidate.

## Candidate Matrix

| V2 Option | User Impact | Build Effort | Delivery Risk | Strategic Fit | Priority Score | Recommended Order |
|---|---:|---:|---:|---:|---:|---|
| SAI concept (guidance-only coach and app guide) | 4 | 2 | 2 | 5 | 5 | 1 (first) |
| Mood system and workflow personalization | 5 | 3 | 2 | 5 | 5 | 2 |
| Holographic Orb internal video layer | 4 | 3 | 3 | 4 | 2 | 2 |
| Live Vocal FX preview module (EQ/Filter/Compressor/Reverb/Delay) | 5 | 4 | 4 | 4 | 1 | 4 |

## Recommendation Snapshot

1. First: SAI concept foundation.
2. Second: Mood system and workflow personalization for high day-to-day UX value.
3. Third: Orb internal video layer for high visual payoff with moderate effort.
4. Fourth: Live Vocal FX preview module due to higher complexity and tuning burden.

## Guardrails

1. Keep SAI guidance-only unless product direction changes.
2. Preserve SAION as user-driven for both sonic and visual control.
3. Re-score this matrix before any V2 implementation kickoff.
